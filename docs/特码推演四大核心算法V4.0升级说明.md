## 二、四大算法详细升级

### 1. 【算法一】热号惯性算法 V4.0

**核心目的**：利用短期开奖惯性，近期开出的生肖有延续出号概率，做正向加权

**技术实现**：
1. ✅ 固定提取最近连续2期特码对应生肖
2. ✅ 区分：上期热肖、上上期热肖，双热肖权重更高
3. ✅ 两期为同一个生肖（连肖）：直接拉满 +15%
4. ✅ 两期不同生肖：统一 +10%
5. ✅ 新增特征：recent2_zodiac 最近两期热肖标签
6. ✅ 新增特征：hot_inertia_coeff 热惯性系数（0.10~0.15）
7. ✅ 连热超过3期后，自动衰减惯性权重，防止追热翻车

**新增特征字段**：
- `recent2Zodiac`: 最近2期生肖列表
- `recent3Zodiac`: 最近3期生肖列表
- `isConsecutiveSame`: 是否连肖
- `consecutiveStreak`: 连肖次数
- `upperHotZodiac`: 上期热肖
- `upperUpperHotZodiac`: 上上期热肖
- `hotLevel`: 热号等级（none/single_hot/double_hot/decayed_hot）

**加权规则**：
- 连肖且为目标生肖：+15% (系数0.15)
- 单期出现：+10% (系数0.10)
- 连热超过3期：自动衰减，最大衰减50%

---

### 2. 【算法二】冷号遗漏回补算法 V4.0

**核心目的**：大数规律下，长期遗漏不出的生肖，存在概率回补修复需求

**技术实现**：
1. ✅ 自定义临界值：默认遗漏＞15期判定为深度冷肖
2. ✅ 分级规则：
   - 轻度遗漏（5~14期）：小幅加分（5-15分）
   - 深度遗漏（≥15期）：大额回补加分（20-25分）
3. ✅ 遗漏期数越大，回补得分线性递增
4. ✅ 触达临界值后，自动激活「冷号回补窗口」
5. ✅ 新增特征：miss_period 当前生肖遗漏期数
6. ✅ 新增特征：miss_repair_coeff 遗漏回补系数
7. ✅ 超极冷肖（遗漏超30期）不盲目拉满，限制最高加分上限，避免死冷不补

**新增特征字段**：
- `missThreshold`: 临界阈值
- `inReboundWindow`: 是否在回补窗口期
- `overflowPeriods`: 溢出期数
- `repairProgress`: 回补进度（0-100%）
- `missLevel`: 遗漏等级（normal/light_cold/deep_cold/extreme_cold）

**分级加分规则**：
- 正常遗漏：0分（超过平均值有轻微加分）
- 轻度遗漏（5-14期）：5-15分（线性插值）
- 深度遗漏（15-29期）：20-25分（线性递增）
- 超极冷肖（≥30期）：12-25分（递减，防止死冷不补）

**回补窗口激活条件**：
- 深度遗漏（≥15期）：激活
- 超极冷肖（≥30期）：激活（但加分递减）

### 3. 【算法三】生肖轮换平衡算法 V4.0

**核心目的**：12生肖不会长期扎堆开出，遵循热转温、温转冷、冷转回补自然轮转

**技术实现**：
1. ✅ 给每个生肖打状态标签：大热肖、温态肖、偏冷肖、极冷肖
2. ✅ 长期霸占开奖的大热肖：逐步扣分
3. ✅ 长期轮空未出的冷肖：逐步累加轮转得分
4. ✅ 新增特征：zodiac_cycle_state 生肖轮转状态
5. ✅ 新增特征：cycle_balance_score 轮转平衡得分
6. ✅ 12生肖循环轮动，保证不会固定只出两三个生肖，贴合真实开奖走势

**新增特征字段**：
- `zodiacCycleState`: 生肖轮转状态（大热肖/温态肖/偏冷肖/极冷肖）
- `cyclePhase`: 轮转阶段（hot_dominant/warm/cold_emerging/cold/balanced）
- `rotationDirection`: 轮转方向（cooling_down/warming_up/balanced/forced_warming_up）
- `recentHotCount`: 近期5期出现次数
- `recentColdCount`: 近期10期冷肖出现次数

**状态标签体系**：
- 大热肖（出现率≥195%平均）：扣5-15分
- 温态肖（150%-195%平均）：扣5分
- 偏冷肖（≤70%平均）：加10-25分
- 极冷肖（≤35%平均）：加15-30分
- 温态肖（正常范围）：加3分

**12生肖轮换平衡检测**：
- 活跃生肖≤5个且最大次数≥3：强制给冷门生肖加权+8分
- 活跃生肖≥10个且最大次数≤2：额外加分+2分

### 4. 【算法四】频次统计排序算法 V4.0

**核心目的**：以历史整体开出频率作为模型基础得分盘口

**技术实现**：
1. ✅ 全周期频次统计：统计自建库以来，每个生肖总开出次数
2. ✅ 统计每个号码总开出次数
3. ✅ 长周期权重：全部历史总频次（默认60%权重）
4. ✅ 短周期权重：近30期近期频次（默认40%权重）
5. ✅ 远近结合动态加权，不唯老数据论
6. ✅ 按综合得分从高到低排序
7. ✅ 筛选Top1独胆、Top3精选特码
8. ✅ 剔除极端异常开奖数据干扰
9. ✅ 频次相近时，交由前面三大算法二次决选

**新增特征字段**：
- `totalFrequency`: 总出现次数
- `longTermFrequency`: 长周期频次
- `shortTermFrequency`: 短周期频次
- `frequencyRatio`: 频次比率
- `isOutlier`: 是否异常值
- `confidenceLevel`: 可信度等级（high/medium/low）
- `needSecondarySelection`: 是否需要二次决选

**动态权重调整规则**：
- 近期数据≥25期：短期权重45%，长期权重55%
- 近期数据15-24期：短期权重40%，长期权重60%（默认）
- 近期数据<15期：短期权重30%，长期权重70%

**异常值检测**：
- 超过2个标准差：标记为异常值，可信度=low
- 超过1个标准差：标记为可疑值，可信度=medium
- 正常范围：可信度=high
- 异常值修正：得分-异常调整值×0.5

**二次决选触发条件**：
- 频次差异<10%时触发
- 由热号惯性、冷号回补、轮换平衡三大算法二次决选

## 三、V4.0终极融合公式

### 总得分计算公式

```
总得分 = (基础频次分 × 动态权重) + 热号惯性加分 + 遗漏回补加分 + 轮转平衡得分
```

### 动态权重公式

```
动态权重 = 1.0 + 热惯性系数 + 遗漏回补系数
```

### 系数范围

- 热惯性系数：0.00 ~ 0.15
- 遗漏回补系数：0.00 ~ 0.80
- 动态权重范围：1.00 ~ 1.95

### 筛选规则

1. **Top1 独胆**：综合得分最高的生肖
2. **Top3 精选特码**：综合得分前三的生肖
3. **频次相近时**：交由前面三大算法二次决选

## 六、使用示例

### 计算某个生肖的综合得分

```javascript
import { calcFourAlgorithmFusion } from './analysis-calc-core.js';

const list = historyData.slice(0, 30); // 取最近30期数据
const result = calcFourAlgorithmFusion(list, '兔');

console.log('总得分:', result.totalScore);
console.log('基础分:', result.baseScore);
console.log('热号惯性:', result.hotInertiaBonus);
console.log('遗漏回补:', result.missRepairBonus);
console.log('轮转平衡:', result.cycleBalanceBonus);
console.log('动态权重:', result.dynamicWeight);
```

### 获取精选生肖排名

```javascript
import { calcSelectedZodiacsV3 } from './analysis-calc-core.js';

const rankedZodiacs = calcSelectedZodiacsV3(historyData, 30);

// Top1 独胆
const top1 = rankedZodiacs[0];
console.log('独胆:', top1.zodiac, top1.totalScore);

// Top3 精选特码
const top3 = rankedZodiacs.slice(0, 3);
top3.forEach((item, index) => {
  console.log(`精选${index + 1}:`, item.zodiac, item.totalScore);
});
```
