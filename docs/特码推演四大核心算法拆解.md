## 📋 四大算法详细说明

### 一、热号惯性算法 (`calcHotInertia`)

### 二、冷号遗漏回补算法 (`calcMissRepair`)
- **核心目的**：大数规律下，长期遗漏不出的生肖存在概率回补修复需求
- **实现要点**：
  - 轻度遗漏（5~14期）：小幅加分 5~15分
  - 深度遗漏（≥15期）：大额回补加分 20~25分
  - 超极冷肖（≥30期）：限制最高加分上限，避免死冷不补
  - 触达临界值后，自动激活「冷号回补窗口」

### 三、生肖轮换平衡算法 (`calcCycleBalance`)
- **核心目的**：12生肖不会长期扎堆开出，遵循热转温、温转冷、冷转回补自然轮转
- **实现要点**：
  - 给每个生肖打状态标签：大热肖、温态肖、偏冷肖、极冷肖

  - 长期轮空未出的冷肖：逐步累加轮转得分 +10~25
  - 12生肖循环轮动，保证不会固定只出两三个生肖

### 四、频次统计排序算法 (`calcFrequencyScore`)
- **核心目的**：以历史整体开出频率作为模型基础得分盘口
- **实现要点**：

  - 长周期：全部历史总频次 × 40分
  - 短周期：近30期近期频次 × 30分

### 二、四大算法详解

#### 算法一：热号惯性 (`calcHotInertia`)
**核心目的**：利用短期开奖惯性，近期开出的生肖有延续概率

**判断逻辑**：
```
提取最近2期特码生肖
    ↓
判断目标生肖是否在这2期中出现
    ↓
连肖（同生肖连续出现2期）→ +15分
单热（只出现1期）→ +10分
    ↓
连热超过3期 → 衰减权重（防止追热翻车）
```

**返回结果**：
- `hotInertiaBonus`: 加分值（0/10/15）
- `hotInertiaCoeff`: 惯性系数（0/0.10/0.15）
- `recent2Zodiac`: 最近2期生肖列表

---

#### 算法二：冷号遗漏回补 (`calcMissRepair`)
**核心目的**：长期遗漏的生肖存在概率回补修复需求

**分级规则**：
```
遗漏期数 < 5期 → 无加分
5-14期（轻度） → +5~15分
15-29期（深度） → +20~25分
≥30期（极冷） → +上限25分，但限制增幅
```

**等级标签**：
- `normal`: 正常
- `light_cold`: 轻冷
- `deep_cold`: 深冷
- `extreme_cold`: 极冷

---

#### 算法三：生肖轮换平衡 (`calcCycleBalance`)
**核心目的**：12生肖遵循热转温、温转冷、冷转热的自然轮转规律

**状态判断**：
```
avgCount = 总期数 / 12

出现次数 ≥ avgCount × 1.3 → 状态：热 → 扣分 -5
出现次数 ≤ avgCount × 0.7 → 状态：冷 → 加分 +10~25
其他情况 → 状态：温 → 加分 +3

额外调整：
- 近期5期出现≥2次 → 额外扣分 -3
- 近期10期冷肖≥7个 → 额外加分 +5
```

---

#### 算法四：频次统计 (`calcFrequencyScore`)
**核心目的**：以历史整体开出频率作为模型基础得分盘口

**双权重模型**：
```
长期权重 60%：
  历史总频次 / 总期数 × 40分

短期权重 40%：
  近30期频次 / 近30期总数 × 30分

基础分 = 60% × 长期分 + 40% × 短期分
```

---

### 三、综合融合公式

```
总得分 = 
  基础频次分 × 动态权重 
  + 热号惯性加分
  + 遗漏回补加分 
  + 轮转平衡得分
```

**详细展开**：
```
动态权重 = 1.0 + 热惯性系数 + 遗漏系数
         = 1.0 + (0~0.15) + (0~0.5)

baseWithWeight = 基础频次分 × 动态权重

totalScore = baseWithWeight + 热惯性分 + 回补分 + 轮转分
           = 基础分 × (1 + 系数) + 加分项
```

```
└─────────────────────────────────────┘
    ↓ (如果候选不足)
┌─────────────────────────────────────┐
│ 第二关：放宽到满足3个条件            │
│ (生肖+尾数+波色) 或 (生肖+尾数+头数) │
└─────────────────────────────────────┘
    ↓ (如果候选仍不足)
┌─────────────────────────────────────┐
│ 第三关：放宽到满足2个条件            │
│ (生肖+尾数) 或 (生肖+波色)           │
└─────────────────────────────────────┘
    ↓
输出TOP5热门号码
```
