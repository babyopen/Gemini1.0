## 二、基础层：四大核心算法

### 1. 热号惯性算法

**核心目的**：利用短期开奖惯性，近期开出的生肖有延续出号概率

**技术实现**：
- 固定提取最近2期生肖作为热肖样本
- 不同热肖：+10%权重
- 连肖同热：+15%权重
- 连热超过3期：触发过热衰减，削减40%热权重

**新增特征**：
- `recent2Zodiac`: 最近2期生肖列表
- `recent3Zodiac`: 最近3期生肖列表
- `isConsecutiveSame`: 是否连肖
- `consecutiveStreak`: 连肖次数
- `hotLevel`: 热号等级

### 2. 冷号遗漏回补算法

**核心目的**：长期遗漏的生肖，存在概率回补修复需求

**技术实现**：
- 轻度遗漏（5~14期）：小幅加分（5-15分）
- 深度遗漏（≥15期）：大额回补加分（20-25分）
- 极冷保护（≥30期）：设置加分上限，规避长期死冷不出

**新增特征**：
- `missPeriod`: 当前遗漏期数
- `inReboundWindow`: 是否在回补窗口
- `repairProgress`: 回补进度（0-100%）
- `missLevel`: 遗漏等级

### 3. 生肖轮换平衡算法

**核心目的**：12生肖遵循热转温→温转冷→冷转回补的轮转逻辑

**技术实现**：
- 自动给12生肖打标签：大热/温态/偏冷/极冷
- 大热肖逐期扣分
- 极冷肖逐期加分
- 12生肖循环轮动检测

**新增特征**：
- `zodiacCycleState`: 轮转状态
- `rotationDirection`: 轮转方向
- `cyclePhase`: 轮转阶段

### 4. 频次统计排序算法

**核心目的**：以历史整体开出频率作为模型基础得分盘口

**技术实现**：
- 长周期权重：60%
- 短周期权重：40%
- 动态权重调整（不唯老数据论）
- 异常值检测和处理
- 频次相近时触发二次决选

**新增特征**：
- `confidenceLevel`: 可信度等级
- `needSecondarySelection`: 需要二次决选
- `isOutlier`: 是否异常值

## 三、识别层：多窗口交叉形态识别（核心升级）

### 窗口配置

```
窗口类型    期数    权重    说明
─────────────────────────────────
短窗口      5期    25%     抓短期拐点
中窗口      8期    50%     主判断窗口（核心）
长窗口      10期   25%     过滤噪音定大趋势
```

### 交叉验证规则

#### 规则1：三窗口形态一致（信号有效，权重拉满）

| 判定结果 | 信号强度 | 权重加成 | 可信度 |
|---------|---------|---------|--------|
| 三热 | 持续热肖 | +20% | 95% |
| 三冷 | 持续冷肖 | +10% | 90% |
| 三温 | 温态肖 | 0% | 85% |

#### 规则2：两窗口一致、一窗口冲突（按多数判定，适度降权）

| 判定结果 | 信号强度 | 权重加成 | 可信度 |
|---------|---------|---------|--------|
| 两热一冲突 | 先热后冷 | -10% | 78% |
| 两冷一冲突 | 先冷后热 | -5% | 78% |
| 两温一冲突 | 震荡轮转 | -15% | 72% |

#### 规则3：三窗口全冲突（无序震荡，优先启用轮转平衡算法）

| 判定结果 | 信号强度 | 权重加成 | 可信度 |
|---------|---------|---------|--------|
| 完全冲突 | 冷热交替 | -25% | 65% |
| 三态全异 | 无序震荡 | -40% | 50% |

### 7种标准走势识别

1. **持续热肖**：三窗口均为热
2. **持续冷肖**：三窗口均为冷
3. **温态肖**：三窗口均为温
4. **先热后冷**：两窗口热、一窗口冲突
5. **先冷后热**：两窗口冷、一窗口冲突
6. **震荡轮转**：两窗口温、一窗口冲突
7. **冷热交替**：三窗口全冲突

### 加权窗口权重计算

考虑中窗口50%权重和各窗口状态：

```javascript
weightedWindowWeight = 
  (短窗口热 ? 25%×1.1 : 短窗口冷 ? 25%×0.9 : 25%) +
  (中窗口热 ? 50%×1.2 : 中窗口冷 ? 50%×0.8 : 50%) +
  (长窗口热 ? 25%×1.05 : 长窗口冷 ? 25%×0.95 : 25%)
```

## 四、融合层：冷热交替行情适配

### 行情特征检测

**冷热交替行情特征**：
- 冷热轮流切换
- 无长连热（hotRatio < 40%）
- 无长期死冷（extremeColdRatio < 30%）
- 温态生肖占比高（warmRatio > 30%）
- 交替频率高（alternationRate > 60%）

### 终极版执行规则

当检测到冷热交替行情时，执行以下适配规则：

#### 规则1：关闭过热衰减限制
```javascript
if (isAlternating) {
  // 关闭连热超过3期的过热衰减
  overheatPenalty = 0;
}
```

#### 规则2：降低激进冷号回补加分
```javascript
if (isAlternating) {
  // 对深度冷肖和极冷肖的回补加分打5折
  if (missLevel === 'extreme_cold' || missLevel === 'deep_cold') {
    adjustedMissRepair = missRepairBonus × 0.5;
  }
}
```

#### 规则3：轮转平衡权重+25%
```javascript
if (isAlternating) {
  // 轮转平衡得分加权25%
  adjustedCycleBalance = cycleBalanceBonus × 1.25;
}
```

#### 规则4：优先选取温态生肖作为主攻方向
```javascript
if (isAlternating && midState === 'warm') {
  // 中窗口为温态时，额外加权10%
  warmPreferenceBonus = totalScore × 0.1;
}
```

## 五、统一最终打分公式（终极版）

### 完整公式

```
总得分 = 
  基础频次分 × 动态权重 
  + 热号惯性加分 
  + 遗漏回补加分 
  + 轮转平衡得分 
  + 多窗口形态加权得分 
  + 温态优先加分（冷热交替时）
  - 过热衰减扣分（非冷热交替时）
```

### 各组成部分详解

#### 1. 基础频次分
```
基础频次分 = 长周期分 × 60% + 短周期分 × 40%
```

#### 2. 动态权重
```
动态权重 = 1.0 + 热惯性系数 + 遗漏回补系数
```

#### 3. 热号惯性加分
```
连肖同热：+15分
单期出现：+10分
连热超3期：衰减40%
```

#### 4. 遗漏回补加分
```
正常遗漏：0分
轻度遗漏（5-14期）：5-15分
深度遗漏（15-29期）：20-25分
极冷（≥30期）：12-25分（递减）
```

#### 5. 轮转平衡得分
```
大热肖：-5~-15分
温态肖：-5~+3分
偏冷肖：+10~+25分
极冷肖：+15~+30分
```

#### 6. 多窗口形态加权得分
```
patternBonus = 基础分 × (windowWeight - 1)
```

#### 7. 温态优先加分（冷热交替时）
```
warmPreferenceBonus = 基础分 × 10%
```

#### 8. 过热衰减扣分（非冷热交替时）
```
overheatPenalty = 热惯性加分 × 40%
```

### 最终得分计算示例

假设某生肖数据：
- 基础频次分：50分
- 动态权重：1.25
- 热惯性加分：10分
- 遗漏回补：15分
- 轮转平衡：8分
- 形态加权：+6分
- 温态优先：+5分（冷热交替时）
- 过热衰减：-4分（非冷热交替时）

```
总得分 = (50 × 1.25) + 10 + 15 + 8 + 6 - 4
       = 62.5 + 35
       = 97.5分（四舍五入：98分）
```
